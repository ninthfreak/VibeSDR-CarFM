Integration spec
New this round — wiring
Station logo (always on): a 62 dp rounded tile immediately left of the PS name in the hero. Bind to the tuned station's logoUri; when null (offline / uncatalogued) show a monogram fallback (callsign initials, or FM before identity resolves) — never blank, per "always display". Don't block render on the logo fetch.
Nearby button: blue-accented pill in the presets header (48 dp). On tap → getNearbyStations() (offline-first, never blocks) and open the picker.
Picker rows: render res.stations in given order (already ranked — don't re-sort by distance). Tap row → tune handler with frequencyMhz*1e6 Hz, mode wfm (same path as presets). Long-press → save-preset flow. Logos fill in on a later call / enrichNow() per visible row.

State encodings (legible without colour — §6)
Stereo / mono — filled ◎ + STEREO (blue) vs outline ○ + MONO (dim). Glyph + word carry it.
Active preset — leading ▶ + blue name + 2 dp border + PLAYING word.
Signal / receivability — face: 12-seg length + dB. Picker: 5 ascending bars (length/position) + explicit km. Amber fill only, never green→red; score is relative rank, so list order is the primary cue.
Nullable picker fields — freq + callsign always; missing logo → no tile (no placeholder clutter); missing city/genre → omitted; missing snapshotDate → footer hidden.
Picker empties — no GPS → "Waiting for GPS…" + manual city pick; DB missing → "Station database not installed yet". Tuned-station logo is the one exception that always renders (monogram fallback).
